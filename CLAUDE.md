# ml-learning

This project is wired into the Leartech hub at `~/leartech/hub/`.

## Hub status (loaded automatically)

@~/leartech/hub/status/ml-learning.md

`PROGRESS.md` (in this repo) is the canonical narrative of what was learned per session — read it alongside the hub status when picking up a session.

## Updating the hub

When significant state changes during this session — a session completed, a new session added, dependency or cluster requirement shifted, a structural convention changed — update `~/leartech/hub/status/ml-learning.md` directly. Keep it concise (it loads into every future session here). Commit and push so other sessions/machines see it.

`PROGRESS.md` updates stay in this repo (per-session learning log with breakpoint line numbers). The hub status is the project-level summary.

Routine in-conversation progress belongs in auto-memory or commit messages, not the hub.
